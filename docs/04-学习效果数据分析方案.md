# 学习效果数据分析方案（答辩可用）

## 1. 目标
通过平台行为数据评估学习效果，回答三个问题：
1) 学习是否持续推进；2) 哪些知识点卡关明显；3) 提示/协作是否提升完成效率。

## 2. 指标设计

### 2.1 学习进度指标
- 总通关数（`totalSolved`）
- 总经验值（`totalExp`）
- 当前等级（`level`）
- 完成率（已通关关卡 / 全部关卡）

### 2.2 学习行为指标
- 总学习时长（`totalStudySeconds`）
- 连续登录天数（`loginStreakDays`）
- 题目重做次数（可基于提交记录统计）

### 2.3 质量指标
- 首次通过率（首次提交通过的关卡占比）
- 平均通过耗时（首次进入到通过）
- 薄弱点分布（`struggleStats` 中 fail/score）

### 2.4 协作指标
- 协作通关数（`collabSolvedKeys`）
- 协作奖励累计（5关 * 20 EXP）
- 协作徽章获得率（“协作大师”）

## 3. 数据来源
- `user_profiles`
- `user_level_submissions`
- `user_badges`

## 4. 建议图表
- 用户完成率柱状图
- 章节卡关热力图
- 提示调用前后通过率对比图
- 协作参与与通关率漏斗图

## 5. 答辩可引用结论模板
- “平台将学习任务拆解为短周期关卡，用户可通过 EXP 与徽章形成正反馈。”
- “提示模块在错误定位与下一步建议上减少了盲目试错时间。”
- “协作任务通过角色分工提高了参与感，并提升了高难关卡通过率。”

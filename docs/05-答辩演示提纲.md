# 答辩演示提纲（10-12分钟）

## 1. 课题背景（1分钟）
- 前端学习痛点：反馈慢、路径不清、实践不足。
- 解决方向：游戏化学习 + AI 实时辅导 + 协作机制。

## 2. 系统架构（2分钟）
- 前端、后端、数据库、实时通信、AI接口。
- 说明模块边界与数据流。

## 3. 核心功能演示（5分钟）
- 登录后进入世界地图，展示关卡推进与奖励反馈。
- 提交代码触发校验、AI提示、通关奖励。
- 展示排行榜、徽章系统。
- 演示双人协作：选关、建房、分工提交、奖励发放。
- 管理员后台：用户管理、题库管理、参考答案辅助。

## 4. 效果与价值（2分钟）
- 学习过程可量化（进度、时长、薄弱点）。
- AI 与协作机制提升体验与完成效率。
- 平台具备继续扩展为教学产品的基础。

## 5. 局限与改进（1-2分钟）
- 当前数据库为 SQLite，后续可迁移 MySQL/MongoDB。
- 增加实验组对照数据，形成更严谨教学效果评估。
- 增加题库管理审计、题目版本化、模型提示评估。
